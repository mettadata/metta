# claude-md-workflow-section-man

## Problem

The `### How to work` section generated into every project's `CLAUDE.md` currently reads:

> For any code change — bug fix, feature, refactor — start with `metta quick <description>` (or `metta propose` for anything non-trivial) before editing files.

This phrasing instructs AI orchestrators to invoke metta CLI commands directly. An AI orchestrator (e.g., a Claude Code session acting as a top-level agent) reading this section concludes that typing `metta quick <description>` into a shell is the correct entry point for a code change. It is not — that entry point is for humans at a terminal.

The correct entry point for an AI orchestrator is the matching `/metta-*` skill (e.g., `/metta-quick`, `/metta-propose`). Each skill wraps the CLI command and also spawns the subagent chain responsible for artifact authoring, review, and verification. When an orchestrator bypasses the skill and calls the CLI directly:

1. The `metta-proposer` subagent is never spawned, so `intent.md` and `summary.md` are authored ad-hoc rather than through the structured subagent flow.
2. Review and verification subagents are never invoked, so low-quality or stub content is never caught before `metta complete` is called.
3. The framework's quality gates are effectively circumvented — `metta complete` accepts any artifact that passes a minimal schema check, with no minimum-content guard on `intent.md` or `summary.md`.

This failure mode is entirely induced by the wording of the generated `CLAUDE.md` text. Fixing the text is the authoritative prevention layer for this class of orchestrator misuse. (A related logged issue — `metta complete` accepting stub artifacts — is a separate, complementary fix.)

The problem exists in three code locations that all emit the same misleading workflow block:

- `src/cli/commands/discovery-helpers.ts` `buildDiscoveryInstructions()` — the `context_template` string (lines 139–147), used by `metta init` scaffolding.
- `src/cli/commands/discovery-helpers.ts` `buildWorkflowSection()` — the authoritative source regenerated by `metta refresh`. (Currently this function does not exist; the template string above is the only source. A dedicated function will be introduced as part of this change so refresh has a single authoritative callsite.)
- `src/delivery/claude-code-adapter.ts` `formatContext()` — the inline workflow block (lines 77–84), used during `metta install` when writing the initial `CLAUDE.md`.

All three emit text that leads AI orchestrators to call CLI commands directly, defeating the skill layer entirely.

## Proposal

Rewrite the `<!-- metta:workflow-start -->` … `<!-- metta:workflow-end -->` block in all three locations to make the skill-first mandate explicit and to forbid direct CLI invocation from AI orchestrator sessions.

The new block MUST:

1. **Mandate skill invocation for AI orchestrators.** Name the three primary entry-point skills with a one-line description of when to use each:
   - `/metta-propose` — for any non-trivial code change (new feature, significant refactor, bug fix with architectural impact)
   - `/metta-quick` — for small, self-contained fixes (typo, config tweak, minor bug with no spec impact)
   - `/metta-fix-issues` — to work through logged issues from `spec/issues/`

2. **Explicitly forbid direct CLI invocation from AI orchestrator sessions.** The block MUST state, unambiguously, that commands such as `metta quick`, `metta propose`, `metta finalize`, and `metta complete` MUST NOT be called directly by an AI orchestrator. CLI invocation is for human terminal users only; skills are for AI orchestrators.

3. **Retain all CLI reference tables** (Lifecycle, Status, Specs & Docs, Organization, System). These tables remain accurate and useful — they document what each skill wraps, and human terminal users need them. They move to a clearly labeled subsection (e.g., `### CLI Reference (human terminal use)`) so the AI/human distinction is structurally visible.

Code locations receiving identical treatment:

- `src/cli/commands/discovery-helpers.ts`: replace the `context_template` workflow block (lines 139–147) with the new text. Extract the workflow block into a standalone exported helper `buildWorkflowSection()` so both `context_template` and the `metta refresh` path share one source of truth.
- `src/delivery/claude-code-adapter.ts`: replace the `formatContext()` workflow block (lines 77–84) with the new text, calling `buildWorkflowSection()` from `discovery-helpers.ts` to avoid duplication.

No other files are changed as part of this spec.

## Impact

- **AI orchestrators** reading a freshly generated or refreshed `CLAUDE.md` will see an explicit mandate to invoke `/metta-quick`, `/metta-propose`, or `/metta-fix-issues` and an explicit prohibition on calling CLI commands directly. This closes the primary path by which orchestrators bypass the subagent chain.
- **Human terminal users** are unaffected. The CLI reference tables remain in place. The new "AI orchestrators must use skills" text does not apply to a human running `metta quick` at a shell — the prohibition is scoped to AI orchestrator sessions, which is stated explicitly in the block.
- **Existing projects** receive the corrected text on their next `metta refresh` run, because `buildWorkflowSection()` is the single authoritative source for the rendered block.
- **No schema changes.** No Zod schema, state file, or config key is modified.
- **No API changes.** `ToolAdapter`, `ProjectContext`, and all public interfaces are unchanged.
- **No behavior changes** to any CLI command. The commands themselves do exactly what they did before; only the text documenting them changes.
- **No new runtime dependencies.**

## Out of Scope

- Changes to the `/metta-quick`, `/metta-propose`, or `/metta-fix-issues` skill files themselves. Skill logic and prompts are not touched.
- Adding enforcement inside `metta complete` to reject stub or placeholder artifact content. That is the subject of the separate logged issue `metta-complete-accepts-stub-placeholder-artifacts-on-intent-.md` and will be addressed in its own change.
- Support for AI tool adapters other than `claude-code-adapter.ts`. At v0.1, Claude Code is the only supported AI tool; other adapters are out of scope.
- Changes to `spec/project.md` (the constitution template) or any other spec template.
- Updating existing archived change summaries or historical documentation to reflect the new wording.
- Automated migration of existing projects' `CLAUDE.md` files. Projects receive the fix through the normal `metta refresh` command; no migration script is provided.
