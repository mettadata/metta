# metta finalize and metta ship are hardcoded to npm/JavaScript gates — blocks any non-JS project from shipping a change. Observed 2026-04-18 on zeus (Rust workspace): after a full clean lifecycle (24 commits, intent/stories/spec/research/design/tasks/implementation/review/summary, 598/598 tests pass, 3 reviewers PASS, 3 verifiers PASS), metta finalize failed on the built-in build gate trying to run `npm run build` — zeus has no package.json. Root cause: gate templates in src/templates/gates/ hardcode npm/npx commands: build.yaml (`npm run build`), lint.yaml (`npm run lint`), tests.yaml (`npm test`), typecheck.yaml (`npx tsc --noEmit`). Only stories-valid.yaml is metta-intrinsic. No project-local override mechanism documented or implemented — .metta/ directory only contains config.yaml with no gates/ subdirectory support. This contradicts the project pitch (zeus CLAUDE.md frames metta as language-agnostic) and blocks adoption on any non-TypeScript/JavaScript project. Fix options: (1) project-local gate overrides at .metta/gates/*.yaml that take precedence over src/templates/gates/, (2) detect stack from .metta/config.yaml and switch default command set, (3) remove hardcoded defaults and require explicit project-level declaration during metta init. Without this, metta cannot ship on Rust, Python, Go, Java, Ruby, etc. Workaround currently requires either adding a package.json stub (band-aid) or bypassing finalize entirely and manually archiving + merging.
