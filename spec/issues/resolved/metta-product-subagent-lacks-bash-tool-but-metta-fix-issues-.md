# metta-product subagent lacks Bash tool but metta-fix-issues skill mandates "Every subagent MUST ... git commit — no exceptions". Reproduced on 2026-04-18 while fixing issue metta-refresh-leaves-claude-md-uncommitted-metta-r: running `metta instructions stories --json --change <name>` returns agent.tools = ["Read", "Write"] only (no Bash). Spawned metta-product subagent wrote stories.md correctly but reported "I don't have a Bash tool available" and asked the orchestrator to run git add/commit. This forces the orchestrator to fall back to manual commits, bypassing the skill's invariant. Source: metta-product agent definition in src/templates/agents/ (likely metta-product.md) — tool list omits Bash. Contrast: metta-proposer persona returned "Read, Grep, Glob" (also no Bash) but the orchestrator skill still says subagents must commit. Either (a) add Bash to metta-product, metta-proposer, metta-executor etc. persona tool lists, or (b) change the skill to make commits the orchestrator's responsibility. Severity: major — silently breaks the skill's stated contract and risks artifacts landing uncommitted if an orchestrator forgets the fallback.
