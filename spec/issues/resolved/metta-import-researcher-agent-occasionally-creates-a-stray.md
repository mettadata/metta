# metta-import researcher agent occasionally creates a stray scaffolded change alongside writing gap files. Reproduced: after running /metta-import on trello-clone demo, the researcher wrote 10 spec/gaps/*.md files and also left behind spec/changes/spec-archaeology-write-gap-files-built-not-documented/ with a .metta.yaml. A subsequent metta finalize failed validate-stories gate with 'Multiple active changes: ... spec-archaeology-write-gap-files-built-not-documented. Specify --change <name>.' Driver had to rm -rf the stray dir before retrying. Fix: metta-import skill or researcher prompt should explicitly forbid calling 'metta propose' during import — gaps go into spec/gaps/ only, never spec/changes/. Also, 'metta import' CLI could warn/reject if a change is mid-flight.
