# metta-fix-issues skill uses wrong CLI command: `metta issue show` (singular) instead of `metta issues show` (plural). Location: src/templates/skills/metta-fix-issues/SKILL.md line 27 reads `metta issue show <issue-slug> --json`. The CLI parses this as `metta issue <description>` where description="show" (singular `issue` only logs new issues, does not take subcommands) and errors: `too many arguments for issue. Expected 1 argument but got 2.`. Same file line 16 correctly uses the plural form `metta issues list --json`. Observed 2026-04-18 on zeus project running `/metta-fix-issues position-db-load-silently-ignored-...`. Claude in the zeus session recovered by self-correcting to the plural form, but only because it noticed the error — the skill itself is broken. Fix: change line 27 to `metta issues show <issue-slug> --json`. Also audit the rest of this skill and any sibling skills for the same singular/plural typo.
