# metta-backlog skill conflicts with metta-guard-edit.mjs hook — backlog item descriptions cannot be written. Observed 2026-04-18 on the metta repo: invoked /metta-backlog add with title+priority+description arguments. The skill instructions say: "The CLI currently stores the title as the description, so after the add, if the user supplied a distinct description, overwrite `spec/backlog/<slug>.md` preserving the frontmatter lines and replacing the body with the new description." After `metta backlog add` succeeded, I attempted to Edit spec/backlog/<slug>.md with the full description body — blocked by .claude/hooks/metta-guard-edit.mjs: "metta-guard: Edit blocked — no active metta change. Start one with /metta:quick <description> or metta quick <description>." Root cause: the guard hook blocks Edit/Write/NotebookEdit/MultiEdit unless (a) an active metta change exists, or (b) the file is in ALLOW_LIST = [spec/project.md, .metta/config.yaml]. spec/backlog/*.md is not on the allow-list, but the skill explicitly tells the orchestrator to edit it. Net effect: backlog items end up with just the title as their description. Fix options: (1) add spec/backlog/*.md, spec/issues/*.md, and spec/gaps/*.md to the ALLOW_LIST so state-mutating metta commands follow-up edits can land, (2) teach the CLI `metta backlog add --description <text>` so no post-add edit is needed and the skill can be simplified, (3) rework the skill to bypass the Edit tool by passing the description directly to a CLI flag. Option 2 is cleanest — the skill would not need a post-add edit at all. Same issue likely affects any future skill that wants to do post-CLI file augmentation.
