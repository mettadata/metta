# Capability folder names polluted with Unicode em dashes (—) because spec-merger does not strip non-ASCII when building capability slugs. src/finalize/spec-merger.ts:48 uses deltaSpec.title.toLowerCase().replace(/\s+/g, '-') which only collapses whitespace. If a spec.md title contains an em dash (e.g. '# Specification — card cover colors'), the em dash is preserved verbatim in the capability slug, producing folder names like specs/specification-—-card-cover-colors/. Evidence: demo trello-clone project has 13 capability directories under spec/specs/ containing a literal em dash (e.g. board-background-colors-board-—-specification, card-labels-small-colored-tags-—-specification). These become the canonical capability identifiers on disk. Fix: align with src/artifacts/artifact-store.ts:12 slugify — .toLowerCase().replace(/[^a-z0-9]+/g, '-').replace(/^-|-$/g, '') — so any non-alphanumeric (em dash, smart quotes, etc.) becomes a single hyphen.
