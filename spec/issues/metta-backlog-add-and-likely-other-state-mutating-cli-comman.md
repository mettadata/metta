# `metta backlog add` (and likely other state-mutating CLI commands) print "Not committed: working tree has other uncommitted tracked changes" when auto-commit is skipped, but do not say which files are blocking. Observed 2026-04-18 on the metta repo: ran `/metta-backlog add ...`, CLI emitted that exact message. User had to run `git status` manually to discover that `.claude/skills/metta-init/SKILL.md` and `.claude/skills/metta-refresh/SKILL.md` had uncommitted modifications. Small fix: include the blocking file list in the skip message, e.g. `Not committed: working tree has 2 uncommitted tracked changes (.claude/skills/metta-init/SKILL.md, .claude/skills/metta-refresh/SKILL.md)`. Saves one round-trip per skip. Likely also affects metta issue, metta refresh (once the refresh-auto-commit fix lands), and any other auto-commit command.
