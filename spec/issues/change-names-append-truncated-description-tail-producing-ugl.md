# Change names append truncated description tail, producing ugly long branch/directory names. This became the git branch name (`metta/fix-issue-position-db-load-silently-ignored-position-db-reco`) and the directory name under spec/changes/. Fix targets: (a) metta-fix-issues skill should pass a shorter/cleaner title to metta propose, or (b) metta propose slug-ification should strip the original issue slug from the tail before truncating, or (c) cap change name length at ~40 chars cleanly at word boundaries.
