# stories-valid gate runs on quick-workflow changes that do not produce stories.md. Observed 2026-04-18 on change fix-two-banner-label-bugs-artifactagentmap-missing-stories-p. Quick workflow artifacts are intent/implementation/verification — no stories stage. But metta finalize runs the stories-valid gate unconditionally, which fails with `not_found — spec/changes/<name>/stories.md`, then skips downstream gates (tests, typecheck) as dependent. Workaround today: hand-write a minimal stories.md with "No user stories — internal change" to pass the gate. Fix options: (1) stories-valid gate should skip (not fail) when the workflow does not declare a stories artifact, (2) gate config should be scoped to workflows that include the stories stage, (3) quick workflow should either skip the gate or write a stub stories.md automatically. Severity: major — blocks finalize on every quick-workflow change that touches banner/label/cosmetic bugs; forces orchestrator workarounds.
