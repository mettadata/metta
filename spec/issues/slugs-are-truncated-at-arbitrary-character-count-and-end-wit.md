# Slugs are truncated at arbitrary character count and end with meaningless fragments (broader than the change-name-specific issue logged earlier as change-names-append-truncated-description-tail-producing-ugl — this covers all slug generation). Observed 2026-04-18 in this session: issue slug `position-db-load-silently-ignored-src-main-rs-723-733-if-pos` (ends with `-if-pos`, a truncation of `if-position-reconciliation-fails`), derived-change slug `fix-issue-position-db-load-silently-ignored-position-db-reco` (ends with `-position-db-reco`, mid-word truncation of `-reconciliation`). Root cause: slug-ification takes the full title and truncates at a fixed character count (~60 chars) with no word-boundary handling. Fix: (1) truncate at nearest word boundary below the limit, (2) prefer slugging only the first-sentence or first-clause of the title, (3) for issue-derived changes, skip concatenating the descriptive tail and just use `fix-<short-issue-slug>`. Cosmetic but pervasive — every branch, directory, and state file carries the ugly slug forever.
