# metta finalize tests gate leaks vitest worker processes on timeout. Observed 2026-04-18 on change centralize-slugify-utility-strip-non-ascii-truncate-at-word: tests gate timed out at its 300000ms limit (test suite legitimately takes ~305s at default concurrency). The gate runner killed the parent npm test process but did NOT reap the orphaned vitest worker node processes, which were adopted by init (PPID 1762) and continued running at 60%+ CPU burning ~4GB RAM each. On the retry, the new finalize spawned its own vitest that had to compete with the orphan for CPU, slowing the retry enough that it ALSO timed out, producing a second orphan. Accumulated two orphans in sequence. Workaround: manually identified the orphans via ps and killed them before retrying finalize. Fix options: (1) gate runner should track and kill child PGID on timeout (process group kill, not just the direct child), (2) gate runner should raise the default timeout for npm test or make it configurable per gate, (3) make vitest explicitly handle SIGTERM and cascade to workers. Severity: major — silently ruins finalize for any test suite close to the 5-min limit, forces manual cleanup, and blocks merge. The underlying test pass/fail is correct (562+/562+) but the gate infra wrongly reports failure.
